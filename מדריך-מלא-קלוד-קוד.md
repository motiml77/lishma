קובץ זה מכיל את **כל** הרצף מקצה-לקצה: הפעלת Firebase, הגירת נתונים, פריסה ל-Vercel,
חיבור התראות מייל. עקוב לפי הסדר. כל שלב חובה אלא אם צוין אחרת.

> ⚠️ **למה החנות לא מציגה ספרים כרגע?**
> הקבצים מחוברים לפרויקט המאוחד `harav-tabady`, אבל בפרויקט הזה **עדיין לא הופעל Firestore
> ועדיין לא הועברו אליו הנתונים**. הספרים נמצאים כרגע בפרויקט הישן `sfarim-tabady`.
> אחרי שלב 1 ושלב 2 למטה — הספרים יופיעו מיד. זו לא תקלה בקוד.

## שלב 1 — הפעלת שירותים ב-harav-tabady (חובה, פעם אחת)

ב-[Firebase Console](https://console.firebase.google.com) → פרויקט **harav-tabady**:

1. **Firestore Database** → **Create database** → בחר **Production mode** → אזור **`eur3`** או **`me-west1`** → Enable.
2. **Storage** → **Get Started** → אשר את הכללים → בחר אותו אזור.

בלי זה תופיע השגיאה: *"The database (default) does not exist for project harav-tabady"*.

---

## שלב 2 — הגירת כל הנתונים (sfarim-tabady ➜ harav-tabady)

מעביר ספרים, הזמנות, הגדרות וכל קבצי ה-Storage, במבנה זהה. **לא מוחק** כלום במקור.

### 2.1 הורדת מפתחות שירות (Service Accounts)
לכל אחד משני הפרויקטים:
- Firebase Console → ⚙️ **Project Settings** → **Service Accounts** → **Generate new private key**.
- שמור את הקבצים בתיקייה `migration/keys/` בשמות המדויקים:
  - `migration/keys/sfarim-tabady.json`
  - `migration/keys/harav-tabady.json`

> 🔒 קבצי המפתח סודיים — לא להעלות ל-Git. `.gitignore` כבר חוסם אותם.

### 2.2 התקנה והרצה
```bash
cd migration
npm install
node migrate.js --dry-run     # סריקה בלבד — בדוק שהמספרים הגיוניים
node migrate.js               # הגירה אמיתית
```
בסיום אמור להופיע: ספרים, הזמנות, הגדרות וקבצים — כולם עם סיכום וספירת שגיאות 0.

### דגלים אופציונליים
| דגל | משמעות |
|-----|--------|
| `--dry-run` | סריקה בלבד, ללא כתיבה |
| `--no-storage` | רק Firestore, בלי קבצים |
| `--no-orders` | בלי אוסף ההזמנות |
| `--skip-existing` | לא לשכתב קיים (ברירת מחדל: שכתוב) |

✅ **אחרי שלב 2 — הספרים כבר יופיעו באתר** (גם בתצוגה מקומית).

## שלב 3 — פריסה ל-Vercel

### דרך א׳ — Vercel CLI (מהיר)
```bash
cd final
npm i -g vercel
vercel login
vercel            # preview
vercel --prod     # production
```
- "In which directory is your code located?" → `./`
- Framework Preset → **Other**. אין build step. אין משתני סביבה.
- תיקיית `migration/` לא נפרסת (חסומה ב-`.vercelignore`).

### דרך ב׳ — GitHub + Vercel Dashboard
1. דחוף את `final/` לריפו GitHub.
2. [vercel.com](https://vercel.com) → **Add New → Project** → ייבא את הריפו.
3. Framework: **Other**, Build Command ריק, Output Directory ריק.
4. **Deploy**.

## שלב 4 — הוספת דומיין Vercel ל-Firebase Auth (חובה לכניסת מנהל)

אחרי הפריסה, קח את כתובת הפרודקשן (למשל `your-site.vercel.app`):
Firebase Console → **Authentication** → **Settings** → **Authorized domains** → **Add domain** → הדבק.
(הוסף גם דומיין מותאם אישית אם תחבר אחד.) בלי זה כניסת המנהל תיכשל.

---

## שלב 5 — התראות מייל בכל הזמנה (Web3Forms)

1. [web3forms.com](https://web3forms.com) → הזן את מייל המנהל → **Create Access Key** → קבל מפתח במייל.
2. באתר החנות → כניסת מנהל → ניהול הזמנות → כפתור **🔑 מפתח שליחת מייל** → הדבק את ה-Key → שמור.
3. (אופציונלי) כפתור **מייל מנהל** → הזן את אותו מייל (משמש כ-reply-to).

כל הזמנה תשלח מייל מסודר (שם הספר, הלקוח, כמות, נקודת איסוף/משלוח, סה"כ, אישור תשלום, מספר הזמנה, תאריך).
כשל בשליחת המייל לא מפיל את ההזמנה — היא תמיד נשמרת ב-Firestore.

> כל Access Key קשור לתיבת מייל אחת (זו שאיתה נרשמת). החלפת מפתח = יעד אחר.
